## üöÄ How to Contribute

### **1. Fork the Repository**

```bash
# Fork on GitHub, then clone your fork
git clone https://github.com/YOUR_USERNAME/ai-career-agent-aws-bedrock.git
cd ai-career-agent-aws-bedrock
```

### **2. Create a Feature Branch**

```bash
git checkout -b feature/your-amazing-feature
```

### **3. Make Your Changes**

- Follow the existing code style
- Add tests for new functionality
- Update documentation as needed
- Include copyright headers in new files

### **4. Test Your Changes**

```bash
# Run locally
npm run dev

# Run tests
npm test

# Check linting
npm run lint
```

### **5. Commit with Proper Attribution**

```bash
git add .
git commit -m "feat: add amazing feature

- Detailed description of changes
- Any breaking changes noted
- References to issues if applicable

Signed-off-by: Your Name <your.email@example.com>"
```

### **6. Submit Pull Request**

- Create a clear PR description
- Reference any related issues
- Include screenshots for UI changes
- Ensure all checks pass

## üìù Contribution Guidelines

### **Code Style**

- Use TypeScript for all new code
- Follow existing naming conventions
- Add JSDoc comments for functions
- Use Prettier for formatting

### **Commit Messages**

Follow conventional commits:

- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation
- `style:` for formatting
- `refactor:` for code restructuring
- `test:` for adding tests

### **Documentation**

- Update README.md for new features
- Add JSDoc comments for functions
- Include examples in documentation
- Update CHANGELOG.md
